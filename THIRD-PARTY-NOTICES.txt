DOOM.BRIX
--------------------------------------------------------------------
1. managed-doom
   - What: the adapted game engine — a hand-translated C# port of
     Linux Doom by Nobuaki Tanaka, incorporating code
     Copyright (C) 1993-1996 Id Software, Inc.
   - Where: src/libs/Doom.Brix.GameEngine (namespaces ManagedDoom.*),
     adapted from commit 9365696e
   - Upstream: https://github.com/sinshu/managed-doom
   - License: GPLv2 "or (at your option) any later version" —
     Doom.Brix/LICENSE_ManagedDoom.txt; the per-file upstream
     copyright headers are preserved in the vendored sources.

2. MeltySynth
   - What: the included SoundFont MIDI synthesizer adapted from work 
     by Nobuaki Tanaka, used for music playback. Adapted into this 
     repository as project Doom.Brix.Synth.
   - Where: src/libs/Doom.Brix.Synth (namespaces Doom.Brix.Synth.*,
     renamed from MeltySynth.* with //was previously: comments on each
     namespace line), vendored from version 2.4.1.
   - Upstream: https://github.com/sinshu/meltysynth
   - License: MIT — Doom.Brix/LICENSE_MeltySynth.txt

3. TimGM6mb SoundFont (TimGM6mb.sf2)
   - What: General-MIDI SoundFont by Tim Brechbill, shipped with the
     application for music playback.
   - Where: src/libs/Doom.Brix.Game/ThirdPartyAssets/TimGM6mb.sf2
     (5,969,788 bytes; md5 1f1ad87ae6f87033d9a591eca567d919; obtained
     from the Debian package timgm6mb-soundfont 1.3-5). Parameter
     tables derived from this SoundFont also ship as Doom.Brix.Synth
     test reference data under
     tests/libs/Doom.Brix.Synth.Tests/ReferenceData/Polyphone/TimGM6mb/.
   - Upstream: https://timbrechbill.com/saxguru/Timidity.php
   - License: GPLv2 — Doom.Brix/LICENSE_TimGM6mb.txt

4. Freeverb reference data (Doom.Brix.Synth tests only)
   - What: numeric test vectors plus one reference impulse-response
     WAV, used to verify the synthesizer's reverb. Generated from
     Freeverb, the public-domain reverb by "Jezar at Dreampoint".
   - Where: tests/libs/Doom.Brix.Synth.Tests/ReferenceData/Freeverb/
   - Upstream: https://github.com/sinshu/freeverb
   - License: public domain.

5. TinySoundFont reference data (Doom.Brix.Synth tests only)
   - What: numeric envelope/LFO reference curves, used to verify the
     synthesizer's envelopes and LFOs. Generated from TinySoundFont by
     Bernhard Schelling.
   - Where: tests/libs/Doom.Brix.Synth.Tests/ReferenceData/TinySoundFont/
   - Upstream: https://github.com/schellingb/TinySoundFont
   - License: MIT.

6. Polyphone (test-data tooling provenance)
   - What: the SoundFont-parameter reference tables in item 3 were
     generated with the Polyphone SoundFont editor. Polyphone itself
     is not distributed here; only the numeric tables it produced are,
     under the TimGM6mb GPLv2 license.
   - Upstream: https://www.polyphone-soundfonts.com/

WOLFENSTEIN.BRIX
--------------------------------------------------------------------
1. csharp-wolfenstein
   - What: adapted rendering and map-decoding code — the CPU
     raycaster, wall/door/sprite renderers, Carmack/RLEW map
     decompression and level-construction logic, by James Randall;
     adapted to this engine and re-pointed at the real .WL1 parsers.
     CODE ONLY: none of that project's pre-extracted asset files
     (PNGs, MP3s) are used or redistributed.
   - Where: src/libs/Wolfenstein.Brix.GameEngine (Rendering/, parts
     of Assets/), vendored at commit accf9db9; each vendored file's
     header cites its upstream source file
   - Upstream: https://github.com/JamesRandall/csharp-wolfenstein
   - License: MIT — Wolfenstein.Brix/LICENSE_CSharpWolfenstein.txt

2. Wolfenstein 3-D iOS source release ("Wolf3D iOS v2.1")
   - What: id Software's official GPL release of the Wolfenstein 3-D
     source (based on the Wolfenstein 3-D Redux codebase by Michael
     Liebscher, incorporating code by DarkOne the Hacker and
     Copyright (C) 1992 Id Software, Inc.). The game logic — actor
     state machines, AI, combat, weapons, doors/areas, pushwalls,
     pickups, level loading and spawn tables — plus the data-file
     format handling (VSWAP/GAMEMAPS/VGAGRAPH/AUDIOT parsers, the
     VGA palette, the AdLib/IMF drivers) are hand-translated from it
     into C#.
   - Where: src/libs/Wolfenstein.Brix.GameEngine (Logic/, parts of
     Assets/ and Audio/), translated from commit d7fff51d; each
     translated file's header cites the original C file
   - Upstream: https://github.com/id-Software/Wolf3D-iOS
   - License: GPLv2 "or (at your option) any later version" —
     Wolfenstein.Brix/LICENSE_Wolf3D-iOS.txt

3. Nuked OPL3
   - What: the cycle-accurate YMF262 (OPL3, OPL2-compatible) FM
     synthesizer emulator by Nuke.YKT, hand-ported to C# for the
     AdLib music and sound effects; the port is verified bit-exact
     against the C original.
   - Where: src/libs/Wolfenstein.Brix.GameEngine/Audio/NukedOpl3.cs,
     ported from upstream version 1.8, commit cfedb09
   - Upstream: https://github.com/nukeykt/Nuked-OPL3
   - License: LGPL-2.1-or-later —
     Wolfenstein.Brix/LICENSE_NukedOPL3.txt (compatible with, and
     distributed here as part of a work under, GPLv2)

Not included: the .WL1 data files (the id Software shareware episode
data) are NOT distributed in this repository or with the application;
the user supplies them through the application's Assets Mode under
id's original shareware terms (see VENDOR.DOC in that download).

Consulted but contributing nothing: the 1992 Wolfenstein 3-D DOS
source release (id Software's "limited use" license, not GPL) was
used strictly as a behavioral reference for data-format facts; no
code from it appears in this repository.
